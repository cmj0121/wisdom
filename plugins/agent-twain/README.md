# Agent Twain Plugin

> Technical Writer agent — user docs, API docs, and migration guides.

Named after Mark Twain — master of clear, concise writing.

Agent Twain is the technical writer of the development team. Twain produces user-facing
documentation, API references, migration guides, and tutorials after QA passes.

## Installation

Install via the wisdom marketplace:

```bash
/plugin install agent-twain
```

## Role in the Team

| Agent             | Role           | Twain's Relationship                    |
| ----------------- | -------------- | --------------------------------------- |
| **agent-smith**   | Project Leader | Assigns documentation tasks             |
| **agent-ward**    | Architect      | Provides design context for docs        |
| **spec-writer**   | Support Tool   | Produces formal specs on Twain's behalf |
| **ascii-grapher** | Support Tool   | Produces diagrams for documentation     |
| **changelog-gen** | Support Tool   | Generates changelogs from git history   |

## How It Works

1. **Understand** — Reads plan, git history, and changed files
2. **Identify** — Determines which docs need creation or updates
3. **Write** — Produces clear, audience-first documentation
4. **Review** — Verifies accuracy against actual code

## Usage

### Magic Words

- `write docs` → Twain documents recent changes
- `document it` → Twain documents the requested feature
- `twain` → Direct invocation
